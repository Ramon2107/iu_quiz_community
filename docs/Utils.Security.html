    Security

    XSS-Schutz Utilities - Sicherheitsfunktionen zum Schutz vor Cross-Site-Scripting

    Dieses Modul bietet umfassende Funktionen zum Schutz vor XSS-Angriffen durch Bereinigung von Benutzereingaben in verschiedenen Kontexten (Text, HTML, Attribute, URLs).

    Enthaltene Funktionen:

    • sanitizeInput: Bereinigt allgemeine Texteingaben
    • sanitizeAttribute: Bereinigt HTML-Attributwerte
    • sanitizeUrlParam: Bereinigt URL-Parameter
    • sanitizeHtml: Bereinigt HTML-Content für sichere Anzeige
    Version:
    • 1.0.0
    Author:
    • Projektteam IU Community Quiz

    (static) sanitizeAttribute(input) → {string}

    Bereinigt HTML-Attribute, um XSS in Attributen zu verhindern

    Diese spezialisierte Funktion ist für die Bereinigung von Werten gedacht, die in HTML-Attributen verwendet werden, wie z.B. in data-Attributen.

    (static) sanitizeHtml(html) → {string}

    Bereinigt HTML-Content für die sichere Anzeige

    Diese Funktion ist für die Bereinigung von HTML-Inhalten gedacht, die direkt in die DOM eingefügt werden, z.B. mit innerHTML. Erlaubt nur sichere Tags: b, i, em, strong, p, br

    (static) sanitizeInput(input) → {string}

    Bereinigt Benutzereingaben von potenziell gefährlichen Zeichen und Code

    Diese Funktion entfernt HTML-Tags und wandelt spezielle Zeichen in ihre HTML-Entitäten um, um XSS-Angriffe zu verhindern. Die Funktion behält dabei die ursprüngliche Bedeutung des Textes bei.
